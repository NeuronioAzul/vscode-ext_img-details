# Internationalization (i18n) Guide

## Supported Languages

Currently, the Image Details extension supports the following languages:

- **English (en)** - Default language
- **Portuguese - Brazil (pt-br)** - Brazilian Portuguese
- **Japanese (ja)** - 日本語
- **Spanish (es)** - Español
- **Chinese Simplified (zh-cn)** - 简体中文

## Architecture

The i18n system uses a **modular architecture** with separate files per locale:

```
src/
├── types/
│   └── index.ts              # Translations interface (defines all keys)
├── i18n/
│   ├── translations.ts       # getTranslations(locale) + fallback logic
│   └── locales/
│       ├── en.ts             # English translations
│       ├── pt-br.ts          # Portuguese (Brazil)
│       ├── ja.ts             # Japanese
│       ├── es.ts             # Spanish
│       └── zh-cn.ts          # Chinese Simplified
```

## How It Works

The extension automatically detects the VS Code language setting (`vscode.env.language`) and displays all UI text in the appropriate language.

## Language Detection Logic

1. First, checks for an exact match (e.g., `pt-br`)
2. If no exact match, checks for the base language (e.g., `pt` from `pt-BR`)
3. Specific fallbacks: Portuguese → `pt-br`, Chinese → `zh-cn`
4. Falls back to English if no match is found

## Adding a New Language

### Step 1: Create the Locale File

Create a new file `src/i18n/locales/<lang-code>.ts` by copying `en.ts`:

```bash
cp src/i18n/locales/en.ts src/i18n/locales/<lang-code>.ts
```

### Step 2: Translate All Keys

Edit the new file. The `Translations` interface in `src/types/index.ts` enforces all keys are present — TypeScript will show errors if any key is missing.

```typescript
import { Translations } from '../../types';

export const fr: Translations = {
    imageDetails: 'Détails de l\'Image',
    fileName: 'Nom du Fichier',
    dimensions: 'Dimensions',
    // ... translate all keys
};
```

### Step 3: Register in translations.ts

Import and add your locale to `src/i18n/translations.ts`:

```typescript
import { fr } from './locales/fr';

export const translations: { [key: string]: Translations } = {
    'en': en,
    'pt-br': ptBr,
    'ja': ja,
    'es': es,
    'zh-cn': zhCn,
    'zh': zhCn,
    'fr': fr,        // Add new locale
};
```

### Step 4: Add Fallback (if needed)

In the `getTranslations()` function, add a fallback for locale variants:

```typescript
if (normalizedLocale.includes('fr')) return translations['fr'];
```

### Step 5: Test

1. Change VS Code language: Command Palette → "Configure Display Language"
2. Select your language and restart VS Code
3. Open an image file and verify all labels are translated

### Step 6: Submit

- Update this document with the new supported language
- Submit a PR to the `dev` branch
