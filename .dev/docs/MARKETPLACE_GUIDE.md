# Marketplace Documentation Guide

This document explains how the extension documentation appears in the VS Code Marketplace.

## 📋 Marketplace Tabs

When users view your extension in the VS Code Marketplace, they see the following tabs:

### 1. **Details** Tab

- **Source**: `README.md` (root)
- **Content**: Main extension description, features, usage instructions, screenshots
- **Status**: ✅ Complete with badges, features, usage guide, and resource links

### 2. **Features** Tab

- **Source**: Extracted from `README.md` "Features" section
- **Content**: List of key features with descriptions
- **Status**: ✅ Complete with comprehensive feature list

### 3. **Changelog** Tab

- **Source**: `CHANGELOG.md` (root)
- **Content**: Version history and release notes
- **Status**: ✅ Created with releases up to v1.3.1

### 4. **Dependencies** Tab

- **Source**: `package.json` → `dependencies` field
- **Content**: Runtime dependencies required by the extension
- **Status**: ✅ Configured with:
  - `exifreader`: ^4.32.0
  - `image-size`: ^1.1.1
  - `jimp`: ^0.22.12

## 📊 Badges

The following badges are configured to appear in the marketplace:

1. **Version Badge** ✅
   - Shows current extension version
   - Auto-updates with each release

2. **Installs Badge** ✅
   - Shows total installation count
   - Updates automatically

3. **Rating Badge** ✅
   - Shows user rating average
   - Updates based on user reviews

**Source**: `package.json` → `badges` array

## 🎨 Gallery Banner

**Configuration**:

```json
"galleryBanner": {
  "color": "#1e1e1e",
  "theme": "dark"
}
```

**Effect**: Sets the background color of the marketplace header to match VS Code's dark theme.

## 🚀 Publishing Command

When ready to publish:

```bash
# Login to marketplace (first time only)
vsce login NeuronioAzul

# Publish new version
vsce publish

# Or publish specific version
vsce publish 0.2.0
vsce publish minor
vsce publish major
```
